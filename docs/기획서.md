# Asset Snap — 서비스 기획서

> "매일 쓰는 귀찮은 가계부는 끝났다. 한 달에 한 번, 내 자산의 위치를 스냅샷으로 찍고 미래를 항해하는 내비게이션"

---

## 1. 서비스 개요

### 1.1 핵심 컨셉
기존 가계부 앱은 **과거 반성**(돈을 어디에 썼는가)에 집중한다.  
Asset Snap은 **미래 준비**(현재 내 자산 체급으로 은퇴·자립 시점이 안전한가)에 집중한다.

| 기존 가계부 | Asset Snap |
|---|---|
| 매일 영수증 입력 | 월 1~2회 계좌 잔액 갱신 |
| 지출 분류·분석 | 자산 스냅샷 + 미래 예측 |
| 과거 회고 | 현재 체급 파악 + 시나리오 탐색 |

### 1.2 핵심 가치
1. **5분 정산** — 계좌별 최종 잔액만 입력하면 끝
2. **자산 체급 파악** — 내 순자산이 또래 대비 어느 위치인지
3. **What-If 시뮬레이션** — 저축률을 10% 높이면 10년 뒤 순자산이 얼마나 달라지는지 실시간 확인

## 2. 핵심 기능 명세

### 2.1 월간 스냅샷 입력 (핵심 기능)
- 자산 / 부채 / 소득 / 지출 4단계 폼
- 각 항목 입력 시 소계·합계 실시간 자동 계산
- 금액 단위: **만원**
- 완료 시 핵심 지표 자동 산출

**입력 항목 구조**
```
자산
├── 금융자산
│   ├── 현금/저축성: 현금/보통예금, CMA, CMA2, 정기적금, 기타
│   ├── 투자자산: 주식/채권, 주식/채권2, 청약저축, 기타
│   └── 보험자산: 정기/상해, 종신, 연금저축, 퇴직연금, 기타
├── 비금융자산(부동산): 주택, 토지/상가, 기타
└── 사용자산(점유): 주거용부동산, 임차보증금, 자동차, 기타

부채
├── 단기부채: 신용카드할부, 마이너스통장, 신용대출, 자동차할부, 임대보증금
└── 중장기부채: 주택담보대출, 전세자금대출, 기타

소득 (월간)
├── 근로소득: 급여, 기타
├── 사업소득: 유튜브, 강의, A/B/C, 기타
├── 양도소득: 주식, 부동산, 기타
├── 이자/배당: 예적금이자, 주식배당, 기타
├── 임대소득: 월세, 기타
├── 연금/보험: 보험금(실비), 기타
└── 기타소득: 인세, 기타

지출 (월간)
├── 저축성
│   ├── 저축/투자: 정기적금, 주식/펀드, 연금
│   └── 부채상환: 대출원금, 기타
└── 소비성
    ├── 고정: 대출이자, 공과금, 관리비, 보험, 월세, 통신비, 정기구독, 기타
    └── 변동: 식비, 생필품, 교육/문화비, 교통비, 의료/미용비, 기타
```

### 2.2 대시보드
- 최신 스냅샷 기준 **핵심 지표 카드** 4개
- **순자산 트렌드** 라인 차트 (전체 이력)
- **자산 구성** 도넛 차트 (현금성/투자/연금/비금융)
- **소득 vs 지출** 바 차트 (월별 비교)
- **목표 달성률** 프로그레스 바 (목표 순자산 대비)

### 2.3 월별 이력 (History)
- 월별 스냅샷 카드 타임라인
- 전월 대비 순자산 증감 (+/- 표시)
- 각 스냅샷 클릭 시 상세 조회
- CSV 내보내기

### 2.4 What-If 시뮬레이션 (v2)
- 월 저축액 / 투자 수익률 / 지출 변수 조정
- 10~30년 후 순자산 곡선 실시간 렌더링
- 시나리오 저장 및 비교

## 3. 핵심 지표 계산식

스프레드시트의 지표를 그대로 구현한다.

| 기호 | 지표명 | 계산식 |
|---|---|---|
| ⓐ | 자산 계 | 금융자산 + 비금융자산 + 사용자산 |
| ⓑ | 부채 계 | 단기부채 + 중장기부채 |
| ⓔ | 순자산 | ⓐ - ⓑ |
| ⓕ | 자기자본비율 | ⓔ ÷ ⓐ × 100 |
| ⓒ | 월소득 계 | 전체 소득 합계 |
| ⓓ | 월지출 계 | 전체 지출 합계 |
| ⓡ | 월 잉여자금 | ⓒ - ⓓ |
| ① | 현금성자산 소계 | CMA + 적금 + 기타 |
| ② | 월 저축/투자 소계 | 정기적금 + 주식/펀드 + 연금 |
| ③ | 월 대출원금 상환 | |
| ④ | 월 대출이자 | |
| ⑤ | 월 소비성지출 계 | 고정소비 + 변동소비 |
| ⓖ | 가계수지지표 | ⑤ ÷ ⓒ × 100 (낮을수록 좋음) |
| ⓗ | 비상예비자금지표 | ① ÷ ⑤ × 100 (6개월치 = 600% 권장) |
| ⓘ | 총부채상환지표 | (③ + ④) ÷ ⓔ × 100 (낮을수록 좋음) |
| ⓙ | 연간 잉여자금 | ⓡ × 12 |
| ⓚ | 연간 저축/투자액 | ② × 12 |
| ⓛ | 연간 대출상환액 | ③ × 12 |
| ⓜ | 연간 자산증가 | ⓙ + ⓚ |
| ⓝ | 연말 예상 자산 | ⓐ + ⓜ |
| ⓟ | 연말 예상 자기자본 | ⓝ - 연말부채 |

## 4. DB 스키마 (Supabase / PostgreSQL)

### 4.1 ERD 요약
```
auth.users (Supabase 기본)
    │
    └── snapshots (1:N)
            │
            └── data (JSONB) — 자산/부채/소득/지출 전체
```

### 4.2 snapshots 테이블
```sql
CREATE TABLE snapshots (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  snapshot_month   DATE NOT NULL,          -- 해당 월의 1일 (2026-06-01)
  data             JSONB NOT NULL,         -- raw 재무 데이터 전체
  -- 집계 컬럼 (대시보드 쿼리 최적화용)
  total_assets     BIGINT DEFAULT 0,       -- ⓐ 자산 계 (만원)
  total_liabilities BIGINT DEFAULT 0,     -- ⓑ 부채 계
  net_worth        BIGINT DEFAULT 0,       -- ⓔ 순자산
  monthly_income   BIGINT DEFAULT 0,       -- ⓒ 월소득 계
  monthly_expenses BIGINT DEFAULT 0,       -- ⓓ 월지출 계
  monthly_surplus  BIGINT DEFAULT 0,       -- ⓡ 월 잉여자금
  created_at       TIMESTAMPTZ DEFAULT NOW(),
  updated_at       TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, snapshot_month)
);

-- RLS 정책
ALTER TABLE snapshots ENABLE ROW LEVEL SECURITY;
CREATE POLICY "users can only access own snapshots"
  ON snapshots FOR ALL
  USING (auth.uid() = user_id);
```

```

```

│  순자산   │ │자기자본비율│ │  월소득   │ │ 월잉여금 │

┌─────────────────────────────┐ ┌──────────────────┐
│   순자산 트렌드 (라인 차트)   │ │  자산 구성 도넛   │
│                              │ │  현금성 ████ 33% │
